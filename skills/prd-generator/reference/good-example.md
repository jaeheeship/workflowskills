# PRD – 게시물 댓글 기능

## Problem

### 현재 상태

사용자가 게시물을 소비한 후 피드백을 남기거나 의견을 표현할 수 있는 공식적인 수단이 없다.

### 문제의 영향

- **사용자 관점**: 게시물에 대한 생각을 공유하거나 질문할 방법이 없어 참여 동기가 낮다
- **비즈니스 관점**: 게시물 소비가 단방향으로 끝나 사용자 간 상호작용이 발생하지 않고, 이는 낮은 재방문율로 이어진다

### 왜 지금 해결해야 하는가

현재 재방문율이 목표 대비 40% 부족하며, 사용자 인터뷰 결과 "다른 사람의 의견을 볼 수 없어 아쉽다"는 피드백이 상위 3개 불만 사항에 포함되었다.

---

## Goal

### 정량적 목표

- 댓글 기능 도입 후 30일 내 댓글 작성률 30% 이상 달성
- 게시물당 평균 상호작용 수 2회 이상 증가

### 정성적 목표

- 사용자가 게시물에 대해 자유롭게 의견을 표현할 수 있는 환경 조성

### 성공 판단 기준

성공 기준은 **기능 구현 완료 여부가 아니라 사용자 행동 변화**로 판단한다.

---

## Target User

### 대상 사용자

- 로그인한 일반 사용자
- 게시물을 읽고 의견을 남기고자 하는 사용자

### 제외 대상

- 비로그인 방문자 (읽기 전용)
- 관리자 계정 (별도 관리 도구 사용)

---

## Core Use Cases

1. **댓글 작성**: 사용자는 게시물 하단에 텍스트 댓글을 작성할 수 있다
2. **댓글 삭제**: 사용자는 본인이 작성한 댓글을 삭제할 수 있다
3. **댓글 조회**: 사용자는 게시물에 달린 댓글 목록을 시간순으로 확인할 수 있다

---

## Scope

### In Scope

- 댓글 작성 (텍스트만)
- 본인 댓글 삭제
- 댓글 목록 조회

### Out of Scope

| 항목             | 제외 이유                               |
| ---------------- | --------------------------------------- |
| 대댓글           | MVP 복잡도 증가, 2차 버전에서 검토      |
| 멘션 기능        | 알림 시스템 의존성, 별도 PRD로 분리     |
| 실시간 알림      | 인프라 추가 필요, 별도 PRD로 분리       |
| 관리자 댓글 관리 | 관리자 도구 PRD에서 다룸                |
| 댓글 수정        | 히스토리 관리 복잡도, 2차 버전에서 검토 |

---

## Priority

| 우선순위 | 기능                        | 근거                                 |
| -------- | --------------------------- | ------------------------------------ |
| Must     | 댓글 작성                   | 핵심 기능, 없으면 PRD 목적 달성 불가 |
| Must     | 댓글 삭제                   | 사용자 통제권 보장 필수              |
| Must     | 댓글 목록 조회              | 상호작용 확인을 위한 필수 기능       |
| Should   | 댓글 작성 후 목록 즉시 갱신 | UX 품질 향상                         |
| Could    | 게시물별 댓글 수 표시       | 참여도 시각화, MVP 이후 고려         |

---

## Decisions & Notes

### 결정 1: 대댓글 미지원

- **결정**: MVP 단계에서는 대댓글을 지원하지 않는다
- **이유**: 복잡도를 낮추고 핵심 상호작용 검증에 집중
- **대안 검토**: 1단계 대댓글만 허용하는 방안 검토했으나, UI/UX 복잡도 증가로 기각

### 결정 2: 댓글 수정 미지원

- **결정**: 댓글 수정 기능은 MVP에 포함하지 않는다
- **이유**: 수정 히스토리 관리, 알림 재발송 여부 등 부가 결정 사항이 많음
- **대안 검토**: 삭제 후 재작성으로 대체 가능

### 결정 3: 익명 댓글 미지원

- **결정**: 모든 댓글은 작성자 정보를 표시한다
- **이유**: 건전한 커뮤니티 문화 조성, 악성 댓글 억제
